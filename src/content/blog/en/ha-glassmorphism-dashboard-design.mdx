After rebuilding three Home Assistant dashboards (desktop, mobile, tablet) with consistent styling, I settled on a glassmorphism design system. Everything is defined by six CSS constants and a handful of card patterns. Here's the complete system so you can copy it directly.

## Why Glassmorphism

Dark home automation dashboards tend to look either clinical (flat Tailwind UI) or visually noisy (too many colored status tiles). Glassmorphism threads the needle: cards feel light and layered, status colors pop against the translucent background, and the blur effect naturally groups related content.

It works especially well against a deep navy background (`#0a0f1a` or similar), where the translucent white glass stands out without high contrast.

## The 6 Core CSS Constants

Every outer card in the system uses exactly these values:

```css
border-radius: 28px;
background: rgba(255, 255, 255, 0.05);
backdrop-filter: blur(15px);
-webkit-backdrop-filter: blur(15px);
border: 1px solid rgba(255, 255, 255, 0.10);
box-shadow: 0 8px 32px rgba(0, 0, 0, 0.3);
```

And one mandatory suppression line:

```css
ha-card::before { display: none !important; }
```

That last line is critical if you're using the **Frosted Glass Dark** theme (or similar). The theme injects a `::before` pseudo-element overlay on every `ha-card`. Without suppressing it, you get double-layering and your custom background is partially obscured.

## The Standard Outer Card Template

In `card_mod`, apply the design system to any card using `custom:stack-in-card` as the outer wrapper:

```yaml
type: custom:stack-in-card
card_mod:
  style: |
    ha-card {
      background: rgba(255,255,255,0.05) !important;
      backdrop-filter: blur(15px) !important;
      -webkit-backdrop-filter: blur(15px) !important;
      border: 1px solid rgba(255,255,255,0.10) !important;
      box-shadow: 0 8px 32px rgba(0,0,0,0.3) !important;
      border-radius: 28px !important;
    }
    ha-card::before { display: none !important; }
cards:
  # your inner cards here
```

## Inner Cards — Transparent Treatment

Cards nested inside a `stack-in-card` should be visually invisible — no background, no shadow, no border. The glass comes from the outer container only:

```yaml
card_mod:
  style: |
    ha-card {
      background: transparent !important;
      box-shadow: none !important;
      border: none !important;
    }
    ha-card::before { display: none !important; }
```

Stack multiple inner cards this way and they appear as a single visual unit inside the glass container.

## Dividers Between Inner Cards

A subtle separator between stacked inner cards:

```css
ha-card {
  border-bottom: 1px solid rgba(255, 255, 255, 0.06) !important;
}
```

Apply this to all inner cards except the last one. Lighter than a standard border — it reads as a content break without interrupting the glass surface.

## Progress Bars via `ha-card::after`

The `::before` pseudo-element is reserved for theme suppression. Use `::after` for progress bars:

```yaml
card_mod:
  style: |
    ha-card {
      overflow: hidden !important;
      position: relative !important;
    }
    ha-card::before { display: none !important; }
    ha-card::after {
      content: '' !important;
      position: absolute !important;
      left: 0;
      top: 0;
      width: {{ states('sensor.battery_level') | int(0) }}% !important;
      height: 100% !important;
      background: linear-gradient(
        90deg,
        rgba(76, 175, 80, 0.18),
        rgba(129, 199, 132, 0.05)
      ) !important;
      pointer-events: none !important;
    }
```

The gradient fades right, so the color intensity tracks the value. This pattern works for:
- Battery percentage (car, robot vacuum, mower)
- Charge level
- Maintenance life remaining (invert: 0% = danger, use red gradient)
- Any 0–100% numeric value

## Map Cards Inside Stack-in-Card

Map cards (vacuum map, camera, etc.) need special handling to work inside a glass container:

```yaml
type: custom:stack-in-card
card_mod:
  style: |
    ha-card {
      background: rgba(255,255,255,0.05) !important;
      backdrop-filter: blur(15px) !important;
      border: 1px solid rgba(255,255,255,0.10) !important;
      box-shadow: 0 8px 32px rgba(0,0,0,0.3) !important;
      border-radius: 28px !important;
      overflow: hidden !important;  /* ← required for corner clipping */
    }
    ha-card::before { display: none !important; }
cards:
  - type: custom:xiaomi-vacuum-map-card   # or any map card
    entity: vacuum.robot
    card_mod:
      style: |
        ha-card {
          border-radius: 0 !important;   /* ← no radius on inner map */
          box-shadow: none !important;
          border: none !important;
        }
```

The `overflow: hidden` on the outer container clips the map's corners to the outer `border-radius`. Without this, the map renders with square corners that bleed outside the glass card. The inner map card gets `border-radius: 0` to avoid double-rounding.

## Badge Suppression

Mushroom cards show colored badges for entity states (e.g., an orange "unavailable" badge on a WiZ light that's temporarily offline). Hide these without affecting the card's main state display:

```yaml
card_mod:
  style:
    ".": |
      mushroom-badge-icon.unavailable {
        display: none !important;
      }
    mushroom-template-card$: |
      .secondary::after {
        content: 'Offline';
        color: rgba(255,255,255,0.4);
        font-size: 0.8em;
      }
```

The `"."` key in the `card_mod` dict targets the host element's shadow DOM context — this is the correct key for mushroom card styling in HA 2024+.

## The Complete Reusable Template

Copy this as a starting point for any new glass card:

```yaml
type: custom:stack-in-card
card_mod:
  style: |
    ha-card {
      background: rgba(255,255,255,0.05) !important;
      backdrop-filter: blur(15px) !important;
      -webkit-backdrop-filter: blur(15px) !important;
      border: 1px solid rgba(255,255,255,0.10) !important;
      box-shadow: 0 8px 32px rgba(0,0,0,0.3) !important;
      border-radius: 28px !important;
    }
    ha-card::before { display: none !important; }
cards:
  - type: custom:mushroom-template-card
    primary: Card Title
    secondary: Subtitle or state
    icon: mdi:home
    card_mod:
      style: |
        ha-card {
          background: transparent !important;
          box-shadow: none !important;
          border: none !important;
        }
        ha-card::before { display: none !important; }
```

## Custom Cards Required

This design system depends on:

- `custom:stack-in-card` — outer glass container
- `custom:mushroom-*` — all inner content cards
- `custom:layout-card` — grid layouts inside glass containers
- `card_mod` — CSS injection (required by all of the above)

**Not required** but used in specific cards: `custom:xiaomi-vacuum-map-card`, `custom:advanced-camera-card`, `custom:apexcharts-card`, `custom:lawn-mower-card`.

Install all via HACS. The `card_mod` integration is a separate HACS install — it's not bundled with mushroom.

## Applying Consistently

The hardest part of a design system in Lovelace is consistency — YAML doesn't have variables. Every card gets its own copy of the CSS block. When I need to change the glass opacity, I change it in 30 places.

My approach: keep a `snippets.yaml` file locally with the full card template as a comment block. When I create a new card, I paste from that file. Not elegant, but it works until HA adds native card templates.

A future improvement would be theming variables (`var(--glass-bg)`) defined once in the HA theme and referenced in `card_mod` styles. This works today if your theme defines custom CSS properties — but requires maintaining a custom theme YAML.
