Danske elspotpriser svinger vildt — somme tider med en faktor 10 mellem den billigste og dyreste time. Home Assistant kan se disse priser i realtid og handle på dem. Sådan automatiserede jeg min Škoda Elroq-opladning omkring de billigste vinduer.

## Energi Data Service-integration

[Energi Data Service](https://www.energidataservice.dk/) er en gratis dansk offentlig API, der leverer timebaserede spotpriser for DK1 (Vestdanmark) og DK2 (Østdanmark). HA-integrationen henter disse direkte — ingen API-nøgle kræves.

Installer via HACS: søg efter **Energi Data Service**, tilføj det derefter via Integrationer. Vælg dit prisområde (DK1 eller DK2) og din energitarifkonfiguration.

Efter opsætning får du:

```
sensor.energi_data_service          — aktuel timepris (DKK/kWh inkl. moms)
sensor.energi_data_service_today    — attributter indeholder alle 24 timepriser
sensor.energi_data_service_tomorrow — morgendagens priser (tilgængelig ~kl. 13:00)
```

Attributstrukturens på today-sensoren ser sådan ud:

```json
{
  "today": [
    {"hour": 0, "price": 0.82},
    {"hour": 1, "price": 0.75},
    ...
    {"hour": 23, "price": 1.43}
  ],
  "min": 0.71,
  "max": 2.18,
  "mean": 1.12
}
```

## Dashboardet — ApexCharts Stepline

Et stepline-diagram er det rigtige format for timebaserede prisdata: hver søjle holder sig stabil i hele timen, derefter hopper den til den næsteværdi. Den aktuelle time fremhæves automatisk.

```yaml
type: custom:apexcharts-card
graph_span: 30h
now:
  show: true
  label: Nu
header:
  show: true
  title: Elpris i dag + i morgen
  show_states: true
  colorize_states: true
series:
  - entity: sensor.energi_data_service
    type: column
    data_generator: |
      return entity.attributes.today.concat(entity.attributes.tomorrow || []).map(h => {
        return [new Date().setHours(h.hour, 0, 0, 0), h.price];
      });
    color_threshold:
      - value: 0
        color: '#4caf50'
      - value: 1.0
        color: '#ff9800'
      - value: 1.5
        color: '#f44336'
```

Grøn under 1,00 DKK/kWh, orange op til 1,50, rød derover. Med et blik kan du se, hvornår du skal tænde opvaskemaskinen eller lade bilen op.

## Find det billigste opladningsvindue

Jeg bruger en skabelonsensor, der finder det billigste 2-timers vindue der er tilbage i dag (eller i nat):

```yaml
template:
  - sensor:
      - name: "Billigste opladningsvindue"
        state: >
          {% set prices = state_attr('sensor.energi_data_service', 'today') %}
          {% if prices %}
            {% set ns = namespace(best_hour=0, best_avg=999) %}
            {% for i in range(prices | length - 1) %}
              {% set avg = (prices[i].price + prices[i+1].price) / 2 %}
              {% if avg < ns.best_avg and prices[i].hour >= now().hour %}
                {% set ns.best_avg = avg %}
                {% set ns.best_hour = prices[i].hour %}
              {% endif %}
            {% endfor %}
            {{ '%02d:00' % ns.best_hour }}
          {% else %}
            ukendt
          {% endif %}
        unit_of_measurement: ""
        icon: mdi:clock-check
```

Dette vises som `"02:00"` eller `"14:00"` — starten på det bedste kommende 2-timers opladningsblok. Et mushroom-template-card på dashboardet viser dette med stor tekst.

## Škoda Elroq-integration

MyŠkoda-integrationen (HACS) forbinder sig til Škodas cloud-API og eksponerer:

```
sensor.elroq_battery_level         — ladetilstand (%)
sensor.elroq_charging_power        — aktuel ladehastighed (kW)
sensor.elroq_range                 — estimeret rækkevidde (km)
binary_sensor.elroq_charging       — lader bilen i øjeblikket?
binary_sensor.elroq_plugged_in     — er kablet tilsluttet?
switch.elroq_charging              — start/stop opladning
```

Switch-entiteten er den vigtige til automatisering — ved at skifte den startes og stoppes AC-opladning fra hjemmeladeren.

## Smart opladningsautomatisering

```yaml
alias: Elroq — Smart opladning på billige timer
trigger:
  - platform: time_pattern
    minutes: "0"
condition:
  - condition: state
    entity_id: binary_sensor.elroq_plugged_in
    state: "on"
  - condition: numeric_state
    entity_id: sensor.elroq_battery_level
    below: 90
action:
  - choose:
      - conditions:
          - condition: numeric_state
            entity_id: sensor.energi_data_service
            below: 1.0
        sequence:
          - service: switch.turn_on
            target:
              entity_id: switch.elroq_charging
      - conditions:
          - condition: numeric_state
            entity_id: sensor.energi_data_service
            above: 1.5
        sequence:
          - service: switch.turn_off
            target:
              entity_id: switch.elroq_charging
```

Kører hver time på klokkeslættet. Hvis prisen er under 1,00 DKK/kWh og bilen er tilsluttet og ikke fuld — oplad. Hvis prisen er over 1,50 — stop. Mellem 1,00 og 1,50 ændres tilstanden ikke (undgår bouncing).

En tilsidesættelses-`input_boolean.elroq_force_charge` omgår priskontrollen på dage, hvor du har brug for fuld opladning uanset hvad.

## Praktiske resultater

Over den første hele måned med smart opladning faldt min Elroqs elomkostning med omtrent 28% sammenlignet med altid-tændt opladning. De største besparelser kommer på dage med store natlige prisforskelle — ikke usædvanligt i Danmark, når vindproduktionen er høj kl. 02–04 og efterspørgslen stiger kl. 08.

Dashboardkortet, der viser aktuel pris, billigste vindue og bilbatteriniveau, er blevet et af de mest tjekkede elementer på min startskærm.
