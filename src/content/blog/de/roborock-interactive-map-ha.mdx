Der Roborock S8 MaxV ist ein leistungsfähiger Roboterstaubsauger — aber die eigentliche Stärke kommt durch die Home Assistant-Integration. Interaktive Raumkarten, zimmerweise Reinigung, Wartungsverfolgung und anwesenheitsbewusste Automatisierungen. So funktioniert alles zusammen.

## Integrationseinrichtung

Die **Roborock**-Integration ist in HA integriert (kein HACS ab HA 2024.x erforderlich). Hinzufügen über **Einstellungen → Geräte & Dienste → Integration hinzufügen → Roborock**, dann mit Roborock App-Anmeldedaten authentifizieren.

Die Integration unterstützt sowohl Cloud- als auch lokales Polling. Ich verwende Cloud-Polling — lokale Kommunikation ist möglich, erfordert aber eine aufwendigere Einrichtung. Für die meisten Nutzer ist Cloud-Polling mit 5-Sekunden-Aktualisierungsintervall in Ordnung.

Nach der Einrichtung:

```
vacuum.roborock_s8_maxv             — Haupt-Staubsaugerentität
sensor.roborock_battery             — Batteriestand in Prozent
sensor.roborock_status              — aktueller Status (reinigt, gedockt, inaktiv, Fehler)
sensor.roborock_last_clean_area     — m² in letzter Sitzung gereinigt
sensor.roborock_last_clean_duration — Minuten in letzter Sitzung
sensor.roborock_total_clean_area    — Gesamt-m² gereinigt
```

Plus Wartungssensoren:
```
sensor.roborock_main_brush_left     — verbleibende Lebensdauer Hauptbürste (%)
sensor.roborock_side_brush_left     — verbleibende Lebensdauer Seitenbürste (%)
sensor.roborock_filter_left         — verbleibende Filterlebensdauer (%)
sensor.roborock_sensor_dirty_left   — Sensorreinigung erforderlich (%)
```

## Interaktive Karte mit xiaomi-vacuum-map-card

`custom:xiaomi-vacuum-map-card` (HACS) rendert die Live-Roborock-Karte in Lovelace mit Raumzonen und direkten Reinigungskontrollen.

```yaml
type: custom:xiaomi-vacuum-map-card
entity: vacuum.roborock_s8_maxv
map_source:
  camera: camera.roborock_map
calibration_source:
  auto: true
internal_variables:
  topic: roborock/map
rooms:
  - id: 16
    name: Wohnzimmer
    icon: mdi:sofa
  - id: 17
    name: Küche
    icon: mdi:silverware-fork-knife
  - id: 18
    name: Schlafzimmer
    icon: mdi:bed
  - id: 19
    name: Büro
    icon: mdi:desk
  - id: 20
    name: Flur
    icon: mdi:door
map_modes:
  - template: vacuum_clean_zone
  - template: vacuum_goto
  - template: vacuum_clean_segment
```

Raum-IDs finden: Den Roborock einmal fahren lassen, dann `state_attr('vacuum.roborock_s8_maxv', 'room_list')` in Entwicklertools → Vorlagen prüfen.

Mit dieser Karte startet das Antippen eines Raums auf der Karte die Reinigung genau dieses Segments. Langes Drücken sendet den Staubsauger zu einer bestimmten Koordinate.

## Wartungsverfolgung

Die Wartungssensoren sind prozentbasiert: 100% bedeutet neu, 0% bedeutet austauschen. Ich stelle diese als farbige Indikatorkarten dar:

```yaml
type: custom:mushroom-template-card
primary: "{{ states('sensor.roborock_main_brush_left') }}%"
secondary: Hauptbürste
icon: mdi:brush
icon_color: >
  {% set pct = states('sensor.roborock_main_brush_left') | int(100) %}
  {% if pct > 40 %}green{% elif pct > 20 %}orange{% else %}red{% endif %}
```

Wenn eine Komponente unter 20% fällt, erhalte ich eine persistente Benachrichtigung:

```yaml
alias: Roborock — Wartungswarnung
trigger:
  - platform: numeric_state
    entity_id:
      - sensor.roborock_main_brush_left
      - sensor.roborock_filter_left
      - sensor.roborock_side_brush_left
    below: 20
action:
  - service: persistent_notification.create
    data:
      title: "Roborock Wartung erforderlich"
      message: >
        {{ trigger.entity_id | replace('sensor.roborock_', '') | replace('_left', '') | replace('_', ' ') | title }}
        ist bei {{ trigger.to_state.state }}%.
      notification_id: roborock_maintenance
```

## Anwesenheitsbewusstes Reinigen

Die praktischste Automatisierung: Staubsauger fahren lassen, wenn ich gehe, andocken bevor ich zurückkomme.

```yaml
alias: Roborock — Reinigen während Abwesenheit
trigger:
  - platform: state
    entity_id: person.rolf
    to: not_home
    for:
      minutes: 5
condition:
  - condition: time
    after: "09:00:00"
    before: "18:00:00"
  - condition: state
    entity_id: vacuum.roborock_s8_maxv
    state: docked
action:
  - service: vacuum.start
    target:
      entity_id: vacuum.roborock_s8_maxv

alias: Roborock — Andocken vor Ankunft
trigger:
  - platform: state
    entity_id: person.rolf
    to: home
condition:
  - condition: state
    entity_id: vacuum.roborock_s8_maxv
    state: cleaning
action:
  - service: vacuum.return_to_base
    target:
      entity_id: vacuum.roborock_s8_maxv
```

Die 5-Minuten-Verzögerung beim Abgangsauslöser verhindert Fehlauslöser durch kurze GPS-Ausreißer.
