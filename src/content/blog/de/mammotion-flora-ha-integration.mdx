Nach drei Jahren mit einem Husqvarna 413c habe ich auf einen **Mammotion Flora** (Luba Mini) gewechselt. Der entscheidende Faktor war nicht nur die Hardware — es war die native Home Assistant-Integration. Kein geschlossenes App-Ökosystem mehr ohne Automatisierungshooks. Hier ist alles, was ich bei der Einrichtung gelernt habe.

## Warum von Husqvarna wechseln?

Der Husqvarna funktionierte mechanisch gut, lebte aber völlig außerhalb meines Smart Homes:

- Keine HA-Integration — nicht einmal über inoffizielle Brücken
- Zeitplanung nur über die proprietäre App
- Kein Wetterbewusstsein — er fuhr fröhlich bei Regen
- Kein Sensor-Feedback zum Auslösen von Automatisierungen

Der Flora hat das alles verändert.

## Der Mammotion Flora auf einen Blick

Der Flora nutzt **GPS + RTK** zur Definition der Mähgrenze — kein Begrenzungsdraht nötig. Die Grenze wird in der Mammotion-App gezeichnet, und der Mäher navigiert innerhalb dieser mithilfe der Satellitenpositionierung. Das Vision Module fügt kamerabasierte Hinderniserkennung hinzu.

Wichtige Spezifikationen meines Geräts:
| Funktion | Detail |
|---------|--------|
| Navigation | GPS + RTK Multi-Satellit |
| Hinderniserkennung | Vision Module (kamerabasiert) |
| Schnittbreite | 26 cm |
| Hangbewältigung | bis zu 45° |
| HA-Integration | Nativ (offiziell) |

## Einrichtung der Home Assistant-Integration

Installiere die **Mammotion**-Integration aus HACS (sie ist inzwischen auch in der offiziellen HA-Integrationsliste verfügbar). Nach der Authentifizierung mit deinem Mammotion-Konto erscheinen folgende Entitäten:

```
lawn_mower.flora          — Haupt-Mäherentität (start, dock, pause)
sensor.flora_battery      — Batteriestand in Prozent
sensor.flora_satellites_robot  — Aktive GPS-Satelliten
sensor.flora_wifi_rssi    — WLAN-Signalstärke (dBm)
binary_sensor.flora_is_mowing — Mähstatus
```

Die `lawn_mower`-Domäne bietet vollständige Service-Unterstützung:

```yaml
service: lawn_mower.start_mow
target:
  entity_id: lawn_mower.flora

service: lawn_mower.dock
target:
  entity_id: lawn_mower.flora
```

## Dashboard-Design

Meine Luba-Ansicht verwendet eine `custom:lawn-mower-card` oben (zeigt Mähergrafik mit Batteriestand) mit Mushroom Chips für Live-Telemetrie:

```yaml
type: vertical-stack
cards:
  - type: custom:lawn-mower-card
    entity: lawn_mower.flora
  - type: horizontal-stack
    cards:
      - type: custom:mushroom-chips-card
        chips:
          - type: entity
            entity: sensor.flora_battery
            icon: mdi:battery
          - type: entity
            entity: sensor.flora_satellites_robot
            icon: mdi:satellite-variant
          - type: entity
            entity: sensor.flora_wifi_rssi
            icon: mdi:wifi
```

Der Satellitanzahl-Chip ist wirklich nützlich — unter 10 Satelliten verschlechtert sich die GPS-Genauigkeit, und der Mäher zögert an Grenzrändern. Ich habe über `card_mod` eine bedingte Farbänderung unterhalb des Schwellenwerts hinzugefügt.

## Wochentags-Zeitplan mit HA-Helfern

Statt auf den Zeitplaner der Mammotion-App zu vertrauen (der nicht auf das Wetter reagieren kann), habe ich einen benutzerdefinierten Zeitplan mit HA-Helfern gebaut:

**Input Booleans** (einer pro Tag):
```
input_boolean.flora_montag
input_boolean.flora_dienstag
input_boolean.flora_mittwoch
input_boolean.flora_donnerstag
input_boolean.flora_freitag
input_boolean.flora_samstag
input_boolean.flora_sonntag
```

**Input Number** für die Startzeit:
```
input_number.flora_startzeit   (min: 7, max: 20, step: 0.5)
```

**Automatisierung:**

```yaml
alias: Flora — Nach Zeitplan fahren
trigger:
  - platform: time_pattern
    minutes: "0"
condition:
  - condition: template
    value_template: >
      {% set day_map = {
        0: 'montag', 1: 'dienstag', 2: 'mittwoch',
        3: 'donnerstag', 4: 'freitag', 5: 'samstag', 6: 'sonntag'
      } %}
      {% set today = day_map[now().weekday()] %}
      {{ is_state('input_boolean.flora_' ~ today, 'on') }}
  - condition: template
    value_template: >
      {{ now().hour == states('input_number.flora_startzeit') | int(0) }}
  - condition: not
    conditions:
      - condition: state
        entity_id: weather.forecast_home
        state:
          - rainy
          - pouring
          - snowy
          - lightning-rainy
action:
  - service: lawn_mower.start_mow
    target:
      entity_id: lawn_mower.flora
```

Läuft jede Stunde zur vollen Stunde, prüft ob heute aktiviert ist, prüft ob es die richtige Stunde ist, und überspringt wenn die Wettervorhersage Regen zeigt.

## Vision Module — Hinderniserkennung

Das Vision Module ist eine kleine Kamera am Mäher-Körper. In meinen Tests vermeidet er zuverlässig:

- Gartenmöbel und Spielzeug
- Auf dem Rasen liegende Schläuche
- Igel (in Dänemark wichtig)

Bei bestimmten Sonnenwinkeln hält er gelegentlich bei Schatten und hellen Flecken an — ein bekannter Firmware-Fehler, den Mammotion kontinuierlich verbessert.

## Ehrliche Bewertung

**Was gut funktioniert:**
- GPS-Grenze ist wirklich drahtlos und zuverlässig über ~12 Satelliten
- HA-Integration deckt alle wesentlichen Befehle und Sensoren ab
- Wetterbewusste Zeitplanung über HA ist eine enorme Verbesserung gegenüber der App
- Batteriestandsmeldung ist genau

**Schwachpunkte:**
- App für die anfängliche Grenzeinrichtung erforderlich — kein Weg darum herum
- Gelegentliche GPS-Drift in der Nähe hoher Hecken lässt den Mäher pausieren
- RTK-Basisstation verbessert die Genauigkeit erheblich, fügt aber Kosten und Komplexität hinzu
- Die HA-Integration braucht manchmal 30–60 Sekunden, um Statusänderungen nach Befehlen zu reflektieren

## Was als Nächstes kommt

Ich plane eine **Satellitanzahl-Prüf**-Automatisierung, die den Mähstart verzögert, wenn die Satellitanzahl zum geplanten Zeitpunkt unter 12 liegt, und nach 10 Minuten erneut versucht. Kombiniert mit der Wetterprüfung macht dies den Zeitplan fast vollständig autonom.

Kommst du von einem Begrenzungsdraht-Mäher, ist der Flora ein echter Fortschritt. Die HA-Integration allein hat den Wechsel lohnenswert gemacht.
